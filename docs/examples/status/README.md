# Status Addon Example

This example demonstrates the optional `a11y-async-button/addons/status` entry.

It shows:

- visible status text synced from core async button state changes
- status target lookup with `data-status-target`
- custom loading, success, and error status text
- reset behavior that restores the original status text

Run `npm run build` before opening the page so `docs/assets/addons/status.js`
exists.
